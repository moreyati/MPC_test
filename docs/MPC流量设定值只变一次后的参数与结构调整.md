# MPC 流量设定值“只变一次、之后几乎不变”问题与后续调参

## 一、当前现象

在拉长 Np、减小 R 之后：

- 流量设定值从个位数增大到了**几百**（说明 Np、R 的调整有效）；
- 但流量设定值**只变了一次**，之后基本不再变化；
- 温度离设定值还有很远，需要流量设定值持续朝目标方向调整才能把温度拉上去。

即：**第一步 MPC 给出了一个“合理”的调整量，但后续步不再继续加大或跟进，设定值卡在第一次的结果上。**

---

## 二、根本原因：优化基准 F0 未随当前工况更新

当前 MPC 在构造优化问题时，**流量始终以固定工作点 F0（如 12500×4）为基准**：

- 决策变量是 **dU**（相对 F0 的流量偏差）；
- 约束写为：`F_abs = F0 + dU[k]`；
- 输出为：`F_sp = F0 + dU[0]`。

因此**每一步优化都是在回答同一个问题**：“从 **12500** 出发，最优的绝对流量是多少？”  
优化器会得到类似 [12800, 12400, 12400, 12400] 的解；下一步仍从 12500 出发，会得到相同或非常接近的解，所以 **F_sp 几乎不变**。

也就是说：**基准没有跟着“当前设定值”或“当前流量”往前走，导致设定值无法逐步累积、只能跳一步就停。**

---

## 三、核心结构调整：用“当前流量/设定值”作优化基准

要让流量设定值能**多步连续调整**，需要让每步优化的基准是**当前工况**，而不是固定的 12500。

### 3.1 思路（不写具体代码，只讲原则）

- **当前做法**：每步都用同一个 F0（如 12500），求的是“相对 12500 的最优绝对流量”。
- **建议做法**：每步用**当前流量设定值**（或当前实际流量）作为基准 F_ref，求的是“相对 F_ref 的增量 dU”，即：
  - 约束中写为：`F_abs = F_ref + dU[k]`，其中 **F_ref = 上一步的 F_sp（或当前 setpoints）**；
  - 输出为：`F_sp = F_ref + dU[0]`。

这样：

- 第 1 步：F_ref = 12500 → 得到 F_sp = 12800；
- 第 2 步：F_ref = 12800 → 得到 F_sp = 13100（在温度仍不足时）；
- 第 3 步：F_ref = 13100 → 可继续增大……  
设定值就能**逐步向目标靠拢**，而不是只动一次。

### 3.2 与状态更新的关系

- 内部状态 **x**（用于预测温度）一般仍按**相对初始平衡点（如 12500）的流量偏差**来更新，即：状态由 `(实际流量 - 初始 F0)` 驱动，这样模型含义不变。
- 只有**优化时的基准**从“固定 F0”改为“当前 F_ref（当前设定值或当前流量）”，这样既能让设定值多步变化，又不破坏状态与温度预测的一致性。

实现时只需在调用 MPC 的 `compute` 前，把“当前流量设定值”传入，并在优化与输出中用其替代原来的 F0 作为 F_ref 即可（具体接口与变量名按你现有代码调整）。

---

## 四、在“已改基准”基础上的参数继续调整

在完成“用当前设定值/流量作基准”之后，若温度跟踪仍偏慢，可以按下面顺序微调。

### 4.1 继续增大输出权重 Q

- **作用**：让“温度跟踪误差”在目标函数里占更大比重，MPC 更愿意用流量变化去追温度。
- **做法**：在现有 Q 上整体放大（如 [10, 10, 10, 2] 或 [20, 20, 20, 5]），或只提高需要重点跟上的回路。
- **注意**：Q 过大可能使流量变化过猛、出现振荡，建议小步加大，观察曲线再定。

### 4.2 再适当减小 R（若仍偏保守）

- **作用**：进一步降低对“单步流量变化”的惩罚，每步允许更大的 dU，设定值变化更积极。
- **做法**：在现有 R 基础上再减（如 0.001、0.0001），观察温度响应与流量平滑度。
- **注意**：R 过小会导致流量抖动、设定值来回晃，需与 Q 一起权衡。

### 4.3 预测步长 Np

- 已拉长到几百后，重点看**预测时长**是否覆盖主要动态（例如 ≥ 2～3 倍主要时间常数）。
- 若过程惯性大、滞后长，可再适当加大 Np，使优化“看得更远”，有利于多步协调、避免只走一步就停的错觉；同时注意 QP 求解时间。

### 4.4 总流量约束 F_total_max

- 若总流量上限卡在 50000 且初始已是 4×12500，MPC 只能做**再分配**（把某路加大、其它路减小）。
- 在“基准已改为当前设定值”后，再分配会多步进行，设定值会逐步移动；若仍觉得“总流量不够用”，可在工艺允许时适当提高 F_total_max（例如 55000～60000），让 MPC 有空间在需要时提高总流量而不仅是内部调配。

### 4.5 单路流量上下限 [0, 20000]

- 若某路需要很大流量才能把温度拉上去，而 20000 的上限过早顶住，会出现“设定值到顶就不再变”的现象。
- 可根据工艺确认是否适当提高单路上限或放宽约束，使 MPC 有足够空间继续加大该路流量。

---

## 五、调参顺序建议

1. **先做结构调整**：把优化基准从“固定 F0”改为“当前流量设定值（或当前流量）”，保证设定值能多步更新。这是解决“只变一次”的关键。
2. **再调 Q**：若跟踪仍慢，优先增大 Q，观察温度曲线与流量曲线是否更跟得上、是否振荡。
3. **微调 R**：若仍偏保守，再适当减小 R；若已出现抖动，则适当增大 R 或减小 Q。
4. **最后看 Np 与约束**：根据过程时间和求解负担调整 Np；根据工艺调整 F_total_max 与单路上下限。

---

## 六、小结

| 现象           | 主要原因                         | 建议方向                                       |
|----------------|----------------------------------|-----------------------------------------------|
| 设定值只变一次 | 优化基准 F0 固定，每步都从 12500 出发 | 用**当前设定值/当前流量**作每步优化的 F_ref   |
| 温度仍离设定远 | 设定值无法多步累积；或 Q/R/Np/约束未配合 | 先改基准，再增大 Q、适当减 R、检查 Np 与约束 |

**核心**：让 MPC 每步从“当前流量/设定值”出发求**增量** dU，再输出 `F_sp = F_ref + dU[0]`，设定值即可多步连续调整；在此基础上再调 Q、R、Np 和约束，即可在“不振荡”的前提下尽量缩短温度跟踪时间。
