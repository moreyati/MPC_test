# 流量曲线与温度曲线时间轴同步方案

## 1. 目标

- **同步时间轴**：流量曲线和温度曲线使用同一套时间基准，且**显示的时间范围一致**，便于对照“同一时刻”的流量与温度。
- **保持变化率**：流量仍按 0.2 s 一个点、温度仍按 5 s 一个点，不改变数据采样与仿真逻辑，只调整**前端显示**。

---

## 2. 当前行为简述

| 项目 | 流量曲线 | 温度曲线 |
|------|----------|----------|
| 数据来源 | `history_flow`，每 0.2 s 一条 | `history_temp`，每 5 s 一条 |
| 时间含义 | 仿真时间 (s)，与后端 `_state["time"]` 一致 | 同上，同一仿真时钟 |
| 点数 | 多（例如 300 点 / 60 s） | 少（例如 12 点 / 60 s） |
| X 轴范围 | 由 Chart.js 按数据或配置决定 | 由 Chart.js 按数据或配置决定 |

后端已经用**同一仿真时间**给两条曲线打时间戳，因此“时间 5 s”在两条曲线里指的是同一时刻。但前端存在两点不足：

- 两个图表的 **X 轴显示范围**（min/max）可能不一致，例如一个显示 0–60 s，另一个显示 0–120 s，或随数据/缩放不同而不同。
- 用户在一个图上**缩放、平移**时间轴时，另一个图不会跟着动，无法保证“同一时间刻度”对齐查看。

因此“同步”主要指：**统一两个图的时间轴显示范围，并（可选）在时间方向上联动缩放与平移**。

---

## 3. 修改后的界面效果

### 3.1 时间轴显示范围一致

- 流量图与温度图的 **X 轴（时间）** 使用**相同的范围**：
  - 例如都显示「当前仿真时间 − 60 s」到「当前仿真时间」（滑动 60 s 窗口），  
    或都显示 0 到「当前仿真时间」（从 0 开始铺满）。
- 同一时刻在两条曲线上的**横向位置一致**：例如 30 s 在流量图和温度图上都在同一竖线位置，方便上下对照。

### 3.2 曲线形态不变（变化率保持）

- **流量曲线**：仍为每 0.2 s 一个点，曲线较密、变化细腻，观感上“变化快”。
- **温度曲线**：仍为每 5 s 一个点，曲线较疏、呈阶梯或折线，观感上“变化慢”。
- 不增加、不插值、不重采样任何一端的数据，仅保证两条曲线画在**同一时间基准和同一时间显示范围**下。

### 3.3 时间轴联动（可选）

- 当用户在**任一图表**上对时间轴进行：
  - **滚轮缩放**（仅时间方向或 X–Y 同时缩放时的时间部分），或  
  - **左键拖拽平移**（仅时间方向或 X–Y 同时平移时的时间部分），  
  则**另一图表的 X 轴**同步为相同的时间范围（min/max）。
- 效果：两个图始终显示**同一段时间**，例如都看 20–80 s，便于对比该时段内的流量与温度。

### 3.4 其他交互不变

- 流量图、温度图的 **Y 轴**（流量值、温度值）仍各自独立：范围、缩放、平移、自动上下限逻辑均不变。
- 滚轮缩放、左键拖拽、双击恢复等交互方式不变；仅增加“时间轴范围一致”以及（若实现）时间方向上的双图联动。

---

## 4. 用户看到的具体表现（修改后）

1. **刚启动仿真**  
   - 两条曲线的 X 轴范围相同，例如都是 0–10 s（或 0–60 s，依实现策略）。  
   - 流量曲线在 0–10 s 内有很多点；温度曲线在 0–10 s 内只有少数点（如 0、5、10 s）。

2. **仿真运行一段时间后**  
   - 例如当前仿真时间 120 s，若采用“最近 60 s”窗口，则两个图都显示 60–120 s。  
   - 流量在 60–120 s 内仍是每 0.2 s 一点；温度在 60–120 s 内仍是每 5 s 一点。  
   - 时间 90 s 在两张图上处于同一竖直位置。

3. **用户缩放/平移时间轴（若实现联动）**  
   - 在流量图上把时间轴放大到 70–90 s，则温度图的时间轴也自动变为 70–90 s。  
   - 或：在温度图上向左拖动时间轴，两张图一起“向左移动”同一段时间范围。

4. **双击恢复**  
   - 若实现联动，可约定：双击任一张图恢复视图时，两张图的时间轴一起恢复为默认范围（例如最近 60 s 或 0 到当前时间）。

---

## 5. 实现要点（思路级，不写具体代码）

1. **统一时间范围来源**  
   - 前端维护一个“当前应显示的时间范围”：`tMin`、`tMax`（或等价的一段时间窗口）。  
   - 每次用 `history_flow` / `history_temp` 和当前仿真时间更新两个图表时，**不再**由各图自己决定 X 轴范围，而是用这一统一的 `tMin`–`tMax` 去设置流量图与温度图的 `scales.x.min` / `scales.x.max`。

2. **时间范围的更新策略**  
   - 例如：`tMax = 当前仿真时间`，`tMin = tMax − 60`（固定 60 s 窗口），或 `tMin = 0`（从 0 开始）。  
   - 若启用“跟随最新”，则随着仿真推进，窗口向右平移，两个图始终显示同一段最近时间。

3. **联动缩放/平移（可选）**  
   - 监听流量图、温度图的 zoom/pan 结束事件（或 scale 变化），读取当前图表的 `scales.x.min`、`scales.x.max`，将另一图表的 X 轴设为相同范围并 `update()`，即可实现时间轴联动。  
   - 注意只同步 X 轴，Y 轴各自独立。

4. **数据与变化率**  
   - 不修改后端采样间隔，不修改 `history_flow` / `history_temp` 的结构；前端只根据现有数据绘制，并统一、可选用上述方式同步 X 轴范围与联动。这样“变化率”完全保持原样。

---

## 6. 小结

| 项目 | 修改后效果 |
|------|------------|
| 时间轴含义 | 两图共用同一仿真时间，同一时刻在同一横向位置 |
| 时间轴范围 | 两图显示相同的时间区间（如 0–当前 或 最近 60 s） |
| 流量曲线 | 仍为每 0.2 s 一点，形态与变化率不变 |
| 温度曲线 | 仍为每 5 s 一点，形态与变化率不变 |
| 时间轴联动（可选） | 任一对时间轴缩放/平移，另一图时间轴同步 |
| Y 轴 | 流量/温度仍各自独立，逻辑不变 |

按上述方式修改后，界面表现为“时间轴同步、变化率保持原样”，便于对照同一时刻的流量与温度。
