# 四回路流量-温度仿真与 MPC 串级控制系统 — 技术说明

## 一、系统概述与功能

本系统是一个**合成氨相关场景下的四回路流量与温度仿真与控制系统**，控制对象采用**内环副对象 + 外环主对象**的串级模型：

- **内环副对象**：**阀门-流量模型 Gp2**（4 路 SISO 一阶惯性），输入为阀门指令 u，输出为 4 路流量 Fcv；**阀门环节 Gv** 对 PID 输出做限幅后送入 Gp2。
- **外环主对象**：**流量-温度模型 Gp1**（4×4 FOPDT），输入为 4 路流量 Fcv，输出为 4 路温度 Tcv。
- **流量内环**：4 个 PID 回路以流量设定值 Fsp 为输入、以 Gp2 输出 Fcv 为反馈，输出 u 经 Gv 限幅后驱动 Gp2。
- **温度外环（可选）**：在 **MPC 串级模式** 下，用户给定温度设定值 Tsp，MPC 计算 Fsp，形成“Tsp → MPC → Fsp → PID → Gv → Gp2 → Fcv → Gp1 → Tcv”的串级结构。

主要功能包括：

- 启停/重置仿真，实时查看流量与温度曲线；
- 两种控制模式：**仅 PID**（手动设定流量设定值）、**MPC 串级**（设定温度，由 MPC 自动给出流量设定值）；
- 在线修改 PID 参数（Kp、Ki、Kd）、流量设定值、温度设定值；
- 流量与温度历史曲线支持滚轮缩放与拖动。

```
└──────────────────────┘
```

| 模块 | 文件 | 作用简述 |
|------|------|----------|
| 后端 / 仿真调度 | `app.py` | Flask 服务、仿真参数与全局状态、仿真循环（PID → Gv 限幅 → Gp2 阀门-流量 → Gp1 流量-温度 + MPC 调用）、REST API |
| 内环副对象（阀门→流量） | `valve_flow_model.py` | Gv：valve_limit 限幅；Gp2：ValveFlowModel 4 路 SISO 一阶惯性，阀门指令 u → 流量 Fcv |
| 外环主对象（流量→温度） | `controlled_model.py` | Gp1：4×4 FOPDT 流量-温度模型，离散化步进，带纯滞后与总流量缩放 |

| 前端 | `templates/index.html` | 单页 UI：状态轮询、设定值/PID 参数输入、流量/温度曲线、启停/重置/模式切换 |

## 三、各模块说明

### 3.1 后端与仿真调度（app.py）

**职责**：定义仿真常数、维护线程安全的全局状态、在独立线程中运行仿真循环、提供 REST 接口供前端读写状态与下发设定值/参数。

**仿真循环逻辑概要**：

1. 若未运行则休眠，否则以 **SAMPLE_TIME** 为步长：
   - 4 路 PID：误差 = 设定值 Fsp − 当前流量 Fcv（Fcv 来自 **Gp2** 上一步输出），得到 u；
   - **Gv**：对 u 做限幅（valve_limit），得到 u_limited；
   - **Gp2**：ValveFlowModel.step(u_limited) 得到本步 Fcv，写回 _state["flows"]；_state["controls"] 为限幅后的阀门指令。
2. 每累积 **PLANT_STEPS_PER_CALL** 个 PID 步（即每 **PLANT_DT** 秒）：
   - 用当前 4 路流量 Fcv 调用 **MIMOFOPDTPlant.step(Fcv)**（Gp1）得到 4 路温度；
   - 用“当前流量 Fcv − MPC 的 F0”更新 **MPC 状态 x**（与 controller_model 一致）；
   - 若为 **mpc_cascade**：以当前温度设定、当前温度、**当前流量设定值 F_ref** 调用 **MPC.compute()**，将返回的 F_sp 写回 `_state["setpoints"]`，供下一段 PID 跟踪。

**关键参数（app.py 内常量）**：

| 参数 | 含义 | 默认/典型值 |
|------|------|-------------|
| NUM_LOOPS | 回路数 | 4 |
| FLOW_MIN / FLOW_MAX | 流量/阀门指令上下限 (Nm³/h) | 0, 60000 |
| INITIAL_SETPOINT_PER_LOOP | 每路初始流量设定值 | 12500（总 50000） |
| SAMPLE_TIME | PID 与 Gp2 步长 (s) | 0.2 |
| PLANT_DT | 流量-温度模型 Gp1 步长 (s) | 3.0 |
| PLANT_STEPS_PER_CALL | 每 PLANT_DT 内 PID 步数 | int(PLANT_DT / SAMPLE_TIME)，如 15 |
| VALVE_FLOW_TAU | Gp2 阀门-流量一阶时间常数 (s) | 8.0 |
| _MAX_HISTORY_FLOW / _MAX_HISTORY_TEMP | 流量/温度历史最大点数 | 600 / 360 |

**状态与历史**：`_state` 中保存 running、setpoints、flows、controls、time、Kp/Ki/Kd、integrals、last_errors、last_times、temperatures、control_mode、T_setpoints；`_history_flow` / `_history_temp` 供前端绘图。

---

### 3.2 内环副对象：阀门-流量（valve_flow_model.py）

**职责**：实现**阀门环节 Gv**（限幅）与**阀门-流量模型 Gp2**（4 路 SISO 一阶惯性），内环 PID 输出 u 经 Gv 限幅后驱动 Gp2，得到实际流量 Fcv。

- **Gv**：`valve_limit(u, u_min, u_max)`，对阀门指令 u 做限幅，与 FLOW_MIN/FLOW_MAX 一致。
- **Gp2**：`ValveFlowModel`，每路动力学为 dFcv_i/dt = (u_i - Fcv_i)/tau_i，离散步长与 SAMPLE_TIME 一致；输出 Fcv 限幅在 flow_min～flow_max。

**主要接口**：

- `valve_limit(u, u_min, u_max)`：返回限幅后的 (4,) 阀门指令。
- `ValveFlowModel(dt, tau, flow_min, flow_max, F0)`：构造 Gp2；`reset(F0=None)` 重置状态；`step(u)` 输入 (4,) 阀门指令，输出 (4,) 流量 Fcv；`get_output()` 返回当前 Fcv。

**关键参数（ValveFlowModel）**：

| 参数 | 含义 | 单位/说明 |
|------|------|-----------|
| dt | 步长 | 秒，与 app 的 SAMPLE_TIME 一致（如 0.2） |
| tau | 一阶时间常数 | 秒，标量或 (4,)，默认 8.0 |
| flow_min / flow_max | 流量输出上下限 | Nm³/h，与 FLOW_MIN/FLOW_MAX 一致 |
| F0 | 初始流量（reset 时状态） | (4,) 默认 [12500,…] |

**调整影响**：tau 增大→阀门-流量响应更慢、流量跟踪更平滑；tau 减小→响应更快、可能更抖。Gv 限幅范围与工艺阀门/流量上下限一致即可。

---

### 3.3 外环主对象：流量→温度（controlled_model.py）

**职责**：实现 4×4 MIMO 流量-温度过程模型（Gp1），供仿真“真实对象”使用。每个通道为 FOPDT：  
ΔT_i = Σ_j G_ij(s) ΔF_j，G_ij(s) = K_ij / (τ_ij s + 1) · e^{-L_ij s}。  
离散化为一阶惯性 + 每通道独立纯滞后缓冲，步长 `dt`（秒），内部用 `dt_min = dt/60`（分钟）与 τ、L 一致。

**主要接口**：

- `__init__(dt=60.0)`：构造模型，离散系数 a、b 与延迟步数由 K、τ、L 计算。
- `reset(T_init=None, F0=None)`：清零内部状态与延迟缓冲，可选初始温度与工作点流量。
- `step(F, noise_std=0.0, enforce_total_flow=None)`：输入 4 路绝对流量 F，输出 4 路温度；可选总流量范围 (min_total, max_total)，超限时按比例缩放四路。

**关键参数（MIMOFOPDTPlant）**：

| 参数 | 含义 | 单位/说明 |
|------|------|-----------|
| dt | 模型步长 | 秒，需与 app 中 PLANT_DT 一致（如 3.0） |
| T0 | 工作点温度 | (4,) 默认 [500,500,500,500] ℃ |
| F0 | 工作点流量 | (4,) 默认 [12500,…] Nm³/h |
| K | 稳态增益矩阵 | 4×4，单位 ℃/(Nm³/h)，当前为 1e-2×[[5,1.2,…],…]，对角约 0.05 |
| tau | 时间常数矩阵 | 4×4，分钟，如 12～25 |
| L | 纯滞后矩阵 | 4×4，分钟，如 0.5～3.0 |

**调整影响**：

- **K 增大**：同流量变化引起更大温度变化，对象更“灵敏”，MPC 更容易用较小流量变化跟温；与 controller_model 的 K 需一致。
- **tau 减小**：响应更快，滞后小，MPC 预测时域内可见效果更明显。
- **L 减小**：纯滞后缩短，温度对流量变化的反应更及时。
- **dt**：必须与 app 中 PLANT_DT 一致，否则仿真与 MPC 步长不匹配。

---

### 3.4 控制器模型：离散状态空间（controller_model.py）

**职责**：将与被控对象相同的 4×4 FOPDT 模型（K、τ、L）离散化为线性状态空间 x_{k+1}=Ax_k+Bu_k，y_k=Cx_k，u=ΔF，y=ΔT；纯滞后用延迟链状态实现。供 **MPC_controller** 做预测，不直接参与仿真步进。

**主要接口**：

- `build_mimo_fopdt_ss(dt=60.0)`：返回 A, B, C, D 和 meta（状态维数、延迟步数等）。  
**重要**：此处 K、τ、L 必须与 **controlled_model** 中一致，否则 MPC 预测与真实对象不一致。

**关键参数**：与 controlled_model 相同的 K、τ、L、dt（秒），以及由之推导的 a、b、delay_steps。无额外可调参数。

---

### 3.5 MPC 控制器（MPC_controller.py）

**职责**：根据当前温度设定 T_sp、当前温度 T_current、当前流量设定 F_ref，在状态空间模型上做有限时域优化，最小化温度跟踪误差与流量增量惩罚，满足单路与总流量约束，只实施第一步控制量，得到新的流量设定值 F_sp = F_ref + dU[0]。内部状态 x 由 app 中根据“实际流量 − F0”同步更新。

**主要接口**：

- `__init__(dt, Np, Q_diag, R_diag, F0, F_total_max)`：构造 MPC，建立 (A,B,C,D)、权重 Q/R、约束。
- `update_state(x_current)`：从仿真侧同步当前状态 x。
- `compute(T_sp, T_current, F_ref=None)`：求解 QP，返回 F_sp；F_ref 为当前流量设定值，用于“本步基准”，使设定值可多步累积。

**关键参数**：

| 参数 | 含义 | 默认/典型 | 调整对系统的影响 |
|------|------|-----------|------------------|
| dt | 与模型/仿真步长一致 | 与 app 的 PLANT_DT 一致（如 3.0） | 与 Plant 不一致会导致预测与对象错位 |
| Np | 预测步数 | 如 80～100 | 增大：预测时长变长，温度跟踪更充分、流量变化更平滑；过大会增加 QP 规模与求解时间 |
| Q_diag | 输出（温度）误差权重，对角 | [5,5,5,1] 等 | 增大：更重视温度跟踪，流量设定变化更积极；过大可能振荡 |
| R_diag | 控制增量（dU）权重，对角 | [0.01～0.1] | 减小：允许每步更大流量变化，温度跟得更快；过小易流量抖动 |
| F0 | 初始平衡点（用于状态更新基准） | [12500]*4 | 需与 Plant 工作点一致，一般不改 |
| F_total_max | 四路流量之和上限 (Nm³/h) | 如 50000 或 80000 | 减小：MPC 只能再分配流量，不能加总流量；增大会允许更高总流量 |

**约束**：单路流量 0～20000 Nm³/h，总流量 ≤ F_total_max。若需更大单路流量，需在代码中修改 20000 上限。

### 3.6 前端（templates/index.html）

**职责**：提供单页 UI，轮询后端状态、下发设定值与 PID 参数、显示流量/温度曲线与实时读数、启停/重置/控制模式切换；曲线支持缩放与拖动，且仅在用户未缩放/拖动时自动按数据调整 Y 轴。

**关键常量与行为**：

| 项 | 含义 | 典型值/说明 |
|----|------|-------------|
| API | 后端基地址 | "" 表示同源 |
| 轮询间隔 | setInterval(poll, 200) | 200 ms 拉取 /api/state |
| FLOW_Y_INIT / TEMP_Y_INIT | 流量/温度图初始 Y 轴范围 | 如 [10000,30000]、[450,550] |
| FLOW_Y_PHYS / TEMP_Y_PHYS | 曲线缩放时的 Y 轴物理上下限 | [0,60000]、[0,600] |
| 首次同步 | setpointInputsSyncedOnce 等 | 仅首次用服务器值填充输入框，之后不覆盖用户输入 |

**调参影响**：轮询间隔越小刷新越频繁、负载略增；Y 轴初值影响初始看图范围，不影响仿真计算。

## 四、关键参数汇总与调整影响

### 4.1 仿真与采样

| 参数 | 所在位置 | 默认/典型 | 调大/调小的影响 |
|------|----------|-----------|------------------|
| SAMPLE_TIME | app.py | 0.2 s | 调大：PID 与 Gp2 更新变慢，动态变“钝”；调小：更细粒度，计算量增 |
| PLANT_DT | app.py | 3.0 s | 调大：Gp1 与 MPC 步长变长，响应更“粗”；调小：更细，MPC 调用更频繁 |
| VALVE_FLOW_TAU | app.py / ValveFlowModel | 8.0 s | 调大：Gp2 阀门-流量更慢，流量跟踪更平滑、略滞后；调小：流量跟踪更快、可能更抖 |

### 4.2 内环副对象 Gp2 与阀门 Gv

| 参数 | 所在位置 | 调整影响 |
|------|----------|----------|
| tau | valve_flow_model.ValveFlowModel | 增大→阀门-流量响应更慢；减小→更快，易抖 |
| flow_min / flow_max | ValveFlowModel | 与工艺/FLOW_MIN/FLOW_MAX 一致；Gv 限幅由 app 中 FLOW_MIN/FLOW_MAX 决定 |

### 4.3 外环主对象（Gp1）与控制器模型

| 参数 | 所在位置 | 调整影响 |
|------|----------|----------|
| K (4×4) | controlled_model + controller_model | 增大→同流量变化产生更大温度变化；两处必须一致 |
| tau (4×4) | 同上 | 减小→响应更快；增大→更慢、惯性大 |
| L (4×4) | 同上 | 减小→滞后小；增大→滞后大 |
| dt | 两处 | 必须与 app 的 PLANT_DT 一致 |

### 4.4 MPC

| 参数 | 所在位置 | 调整影响 |
|------|----------|----------|
| Np | MPC_controller.py | 增大→预测更长、跟踪更好、计算更重；过小会导致“只动一步就停”或响应不足 |
| Q_diag | MPC_controller.py | 增大→更重视温度误差，流量设定变化更积极；过大易振荡 |
| R_diag | MPC_controller.py | 减小→每步允许更大流量变化；过小易流量抖动 |
| F_total_max | MPC_controller.py + app 构造 | 减小→只能再分配流量；增大→允许更高总流量（若工艺允许） |
| 单路上下限 0/20000 | MPC_controller.py 约束 | 提高上限→允许单路更大流量，需与工艺一致 |

### 4.5 结构相关（已实现）

- **F_ref 每步传入当前流量设定值**：使 MPC 每步从“当前设定值”出发求增量，流量设定值可多步累积，避免“只变一次就停”。若去掉或传错，会退回“只从 F0 出发”的行为。

## 五、数据流与调用关系简要

1. **前端**：每 200 ms 请求 `/api/state`，得到 running、setpoints、flows、temperatures、T_setpoints、Kp/Ki/Kd、control_mode、history_flow、history_temp；用户修改设定值/参数时 POST 到对应 API。
2. **仿真线程**：  
   - 每 SAMPLE_TIME：读 setpoints、flows(Fcv)、Kp/Ki/Kd 等 → 算 PID 得 u → Gv 限幅得 u_limited → Gp2.step(u_limited) 得 Fcv → 写回 _state["flows"]、_state["controls"]=u_limited、time、integrals 等，并追加 history_flow。  
   - 每 PLANT_DT：用当前 flows（Fcv）调 _plant.step(Fcv)（Gp1）→ 得到温度，更新 _state["temperatures"]、history_temp；用 Fcv − F0 更新 _mpc_state_x；若 mpc_cascade，用当前 T_setpoints、temperatures、setpoints 作 F_ref 调 _mpc_controller.compute()，将返回的 F_sp 写回 _state["setpoints"]。
3. **MPC**：使用 controller_model 的 (A,B,C,D)，在每步用 F_ref、当前状态 x、T_sp、T_current 求解 QP，输出 F_sp = F_ref + dU[0]。
